## Bug DI — DurableTemporalTransportFactoryPass

- **Erreur** : `TemporalTransportFactory::__construct(): Argument #4 ($workflowRegistry) must be of type ?WorkflowRegistry, InMemoryEventStore given`.
- **Cause** : `DurableTemporalTransportFactoryPass.process()` injectait `EventStoreInterface` à l'index 3, vestige d'une ancienne signature du constructeur. Or l'argument #4 est `?WorkflowRegistry` (nécessaire à `TemporalJournalTransport::fromConnection()`).
- **Fix** : `$def->setArgument(3, new Reference(EventStoreInterface::class))` → `new Reference(WorkflowRegistry::class)`.
- **Test de régression** : `DurableTemporalTransportFactoryPassTest` (6 tests) ajouté dans `tests/unit/DurableBundle/DependencyInjection/Compiler/`.

## Bug Temporal — workflows non déclenchés dans Temporal

- **Symptôme** : avec `durable.temporal.dsn` configuré, démarrer un workflow ne créait aucune exécution dans Temporal.
- **Cause** : suite à la suppression de `TemporalJournalEventStore` (qui appelait `SignalWithStartWorkflowExecution` sur `append(ExecutionStarted)`), aucune classe n'appelait plus `WorkflowClient::startAsync()` (`StartWorkflowExecution` gRPC). `DurableExtension.registerWorkflowMessengerServices()` enregistrait toujours `MessengerWorkflowResumeDispatcher` (dispatch Messenger), même en mode Temporal natif. Or `TemporalJournalTransport` est receive-only et lève `LogicException` sur `send()`.
- **Fix** :
  1. Création de `src/Bridge/Temporal/Port/TemporalWorkflowResumeDispatcher` :
     - `dispatchNewWorkflowRun()` : sauvegarde les métadonnées + appelle `WorkflowClient::startAsync()`.
     - `dispatchResume()` : no-op (Temporal re-programme lui-même le prochain workflow task).
  2. `DurableExtension.registerWorkflowMessengerServices()` détecte `isTemporalNative` :
     - Temporal natif → enregistre `TemporalWorkflowResumeDispatcher`, n'enregistre PAS `ResumeWorkflowHandler` (inutile et potentiellement confondant).
     - In-memory → comportement précédent (`MessengerWorkflowResumeDispatcher` + `ResumeWorkflowHandler`).
- **Tests** : 44/44 Symfony verts après le changement.
